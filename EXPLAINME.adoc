The README makes claims. This file backs them up with code paths, caveats, and honest gaps.

== Claim 1: Consolidated Monorepo — 52 Repos Absorbed, Sprawl Eliminated

[quote, README]
____
A consolidated monorepo of static site generator experiments and implementations from the
hyperpolymath ecosystem. This monorepo was consolidated from 52 individual *-ssg repositories
plus ssg-fixes on 2026-02-08 to reduce repository sprawl.
____

*How it works:* On 2026-02-08, 52 standalone `*-ssg` GitHub repositories and the `ssg-fixes` repo
were merged into this monorepo. The three-tier taxonomy (`implementations/`, `variants/`, `stubs/`)
sorts them by maturity: implementations have 7+ source files (minimum viable standalone SSG logic),
variants have 2-7 (experiments and smaller ideas), and stubs have 0-1 (RSR scaffolding with no
content yet). The 7 implementations are: `rats` (17 files), `odd` (15), `eclipse` (15),
`wokelang` (13, Rust/Cargo), `hackenbush` (11), `pharos` (11), `qed` (8). This consolidation
reduces CI costs, mirrors, and registry entries by ~52x for the stub/variant tier. The two
canonical SSG frameworks (`poly-ssg` and `casket-ssg`) remain as separate repositories because
they are production-grade frameworks rather than experiments.

*Honest caveat:* "Absorbed" means the files were moved in; it does not mean the SSGs were
evaluated, tested, or brought to any common standard. Most stubs and variants are in the state
they were in when the originating repo was last touched — which may be months-old scaffolding.
There is no cross-collection test runner that exercises all implementations.

== Claim 2: Language Diversity — Rust, Gleam, and Deno-Targeted SSGs Present

[quote, README]
____
Full SSG implementations with meaningful source code: rats (17 files), odd (15), eclipse (15),
wokelang (13, Rust), hackenbush (11), pharos (11), qed (8). Variants: sparkle (Gleam), zigzag,
wagasm.
____

*How it works:* `implementations/wokelang/` contains a `Cargo.toml` workspace and Rust source, making
it the only implementation in a compiled systems language. `variants/sparkle/` uses Gleam's BEAM
target. The Deno-targeted SSGs use `deno.json` imports rather than `package.json`. The
`implementations/rats/` and `implementations/eclipse/` directories contain the most source files
(17 and 15 respectively) and represent the most complete feature sets within the collection.
`ssg-fixes/` (also absorbed) contains cross-cutting fixes that were being applied to multiple
SSGs before the monorepo consolidation.

*Honest caveat:* Language policy compliance is uneven across the collection. Some stubs contain
`package.json` artifacts from pre-2026 development when TypeScript and npm were still in use.
The `npm-bun-blocker` and `ts-blocker` CI workflows enforce the policy going forward, but older
files in stubs and variants have not been audited or cleaned up.
