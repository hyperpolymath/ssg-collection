== Why 60-ssg?
=== Lexical Scope Rigour
ALGOL 60 introduced the code block (`begin`...`end`). In `60-ssg`, site variables are strictly scoped to their blocks, preventing the global-state corruption common in modern scripting engines.

=== Algorithmic Clarity
Used as the standard for algorithm publication for decades, this engine provides a level of readability and discipline that prioritises stability over alpha features.

== Features
* **Strict Block Scoping** - Variables restricted to local execution units.
* **Podman-First** - Isolated runtimes for legacy block-logic.
* **Accessibility** - Integrated support for **BSL**, **GSL**, and **Makaton**.

== Requirements
* **ALGOL 60 Interpreter**
* **Just** (Orchestrator)
